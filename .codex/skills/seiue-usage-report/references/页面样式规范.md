# 页面样式规范

## 整体风格

- 使用内联 CSS 生成单文件 HTML。
- 严格遵循高级商业咨询报告的文档排版规范，优先接近麦肯锡、四大这类正式研报的阅读体验。
- 生成页面样式前，先用以下公式明确视觉提示词：`平台 + 主题 + 视觉风格 + 颜色 + 细节/组件`。
- 默认样式提示词：`单文件 HTML 商业咨询报告 + 希悦使用情况深度洞察 + 顶级咨询研报级极简文档流 + 纯白背景、深炭黑正文、沉稳商务色图表 + 三线表、底部实线章节标题、白底图表、斜体图表解读、Tailwind CSS 和 Lucide 图标`。
- 可以使用 Tailwind CSS 的工具类思路组织间距、排版和响应式规则；如果为了单文件交付引用 CDN，必须确保报告在无网络时仍保留可读的内联 CSS 基础样式。
- 可以使用 Lucide 图标作为极少量线性辅助图标；图标只能服务于章节标识或提示说明，必须保持黑白或深灰线性风格，不得产生互联网仪表盘感。
- 页面背景必须为 `#FFFFFF`。
- 页面内容居中，最大宽度约 `900px`。
- 页面上下留白建议为 `padding: 40px 0;`，模拟 A4 纸质报告阅读体验。
- 摒弃卡片式设计：不要使用 `box-shadow`，不要使用大圆角；`border-radius` 必须为 `0` 或极小的 `2px`。
- 不要使用包裹正文内容的大面积背景色块，不要把章节做成互联网风格的卡片墙。
- 参考“宁波赫德实验学校”PDF 的正式报告感：A4 竖版阅读节奏、白底、编号章节、图表块和分析 bullet 穿插。
- 不要求逐像素复刻 PDF，必须根据每次真实数据的模块覆盖和重点结论优化章节密度、图表数量和页面长度。

## 字体与文本

默认字体策略：

```css
h1, h2, h3 {
  font-family: Georgia, "Times New Roman", serif;
}

body, p, li, table {
  font-family: Helvetica, Arial, sans-serif;
}
```

- 正文颜色必须是高对比度深炭黑，如 `#111827` 或 `#1C1C1C`。
- 正文行高建议为 `1.8`。
- 首行不缩进，段落间距明确，建议 `margin-bottom: 20px;`。

重点结论使用：
- `<strong>`
- `<mark>`
- 审慎、克制的文字强调

不要使用过度装饰、强烈渐变或影响打印阅读的深色背景。

## 标题层级

- `h1` 居中、粗体、字号大，下方保留充足空白，体现学校名称和报告主题。
- `h2` 用于一级章节，可采用 `1. 报告说明`、`2. 学校概览`、`3. 模块使用概况` 这类编号标题。
- `h2` 必须使用底部实线作为章节分隔，如 `border-bottom: 2px solid #000000;` 或深蓝色实线，不能用色块横条代替。
- `h3` 用于模块或子问题。
- 标题之间通过留白和分隔线保持层次清晰。
- 页面字号层级参考：主标题约 24-28px，一级章节约 20-22px，二级标题约 17-18px，正文约 14-16px。

## 表格样式

- 表格是按需组件，不是每个模块的固定版式；只有业务判断需要时才渲染 `.table-block`。
- 必须采用三线表规范，不使用全封闭网格线。
- 隐藏所有垂直边框，不要保留 `border-left` 和 `border-right`。
- 仅保留表格顶部粗线、表头底部细线、表格底部粗线。
- 表格内部不要使用交替背景色，不要使用斑马纹。
- 数字列必须右对齐，文本列左对齐。
- 移动端允许横向滚动。
- 不要为了视觉统一给每个章节都补一张三线表；没有新增信息的表格应直接省略。

## 图表样式

- 图表统一使用 ECharts 生成，并显式指定 SVG 渲染器：`echarts.init(dom, null, { renderer: 'svg' })`。
- 最终 HTML 直接内嵌 ECharts 容器、初始化脚本和 option，保持单文件 HTML 交付；不创建额外的 HTML 生成脚本；图表统一走 SVG 渲染路径，不使用 canvas。
- 图表必须是白底，不能带灰色背景或外边框。
- 只保留必要的浅灰色水平参考线，隐藏不必要的网格线和边框。
- 图例和坐标轴字体必须与正文风格一致。
- 图表配色应年轻化、清爽且专业，优先使用 `#39B981`、`#265BEA`、`#F59E11`、`#8554F6`、`#3CB6D4` 作为主色板，再按需要扩展相邻色；禁止高饱和度荧光色、整页灰暗配色和过于黑暗的主色。
- 图表必须包含标题、完整图例、坐标轴标签和必要的数据标签。
- 折线图默认显示峰值标签；如果同图存在多条核心折线，则每条核心折线至少显示一个峰值标签。峰值用 ECharts `markPoint` 呈现，`symbol` 使用水滴形 `pin`，标签内容只显示具体数值，例如 `128`，不附加“峰值”“次”“人”或说明文字。
- 峰值 `markPoint` 推荐写法：`markPoint: { symbol: 'pin', symbolSize: 46, data: [{ type: 'max' }], label: { formatter: p => String(p.value) } }`；如果数值较长，可适当增大 `symbolSize`，但保持水滴形标记轻量克制。
- 图例必须放在不遮挡数据的位置，必要时放到底部横向排布、右侧纵向排布或拆分成多行，确保系列名称显示完整。
- 当图例文字较长或系列较多时，必须增加图例项之间的水平间距、垂直间距、列间距和图例留白，避免图例文字重叠、挤压或被图形遮挡。
- 图表类型必须根据实际业务数据单独选择，不要让多个模块重复使用同一种图表形态。
- 柱状图适合模块对比、角色对比和阶段量级对比。
- 堆叠柱状图适合展示结构变化、角色构成和分项叠加总量。
- 双轴柱线组合图适合同时展示总量与活跃人数、任务布置与批阅、操作量与覆盖率等双指标关系。
- 多序列折线图适合时间趋势、学期节奏、角色活跃和并行流程的联动变化。
- 同一模块存在多个角色、多个状态或多个并行口径时，优先使用叠加折线图或多序列折线图，把多条序列放在同一时间轴上比较。
- 分面小图适合类目很多但每个类目都需要完整展示时使用。
- 环形图只在类别较少且需要突出占比关系时使用；类别过多时改用排序条形图或堆叠图。
- 没有时间维度时不要生成折线图；当类别过多导致环形图拥挤时不要硬用占比图。
- 当一个结论同时涉及结构、总量与趋势时，优先使用组合图或叠加图，而不是拆成多个低信息密度的小图。
- 必须确保全部数据可见：标签过长时换行、旋转或缩写并配完整图例；序列过多时拆分分面或改表格，不得出现遮挡。
- 右侧最后一个数据点、最后一组柱体、最后一个刻度文字和末端标签必须完整可见，不得贴边裁切。
- 当横轴内容过密时，优先增加右侧边距和底部留白；如仍拥挤，再压缩横轴类目间距、减小 `bargap`、降低刻度标签密度、旋转标签或拉宽画布。
- 横轴拥挤时，不要用整体缩小图片比例的方式解决；应优先压缩单个类目间距、柱间距和标签密度，同时保持图表主体清晰可读。
- 即使降低横轴文字标签密度，也不能丢失真实数据点；所有时间点和类目数据都必须完整绘制。
- 禁止用固定长度切片直接截断序列，例如 `sorted(points)[:12]`、`points[:12]`、`x_values[:12]`；如果需要聚焦最近区间，必须先得到用户明确授权，或在图中清楚标明“仅展示最近 N 个点”。
- 如果标签重叠严重，折线图至少保留水滴形峰值标签，其余再按末值、异常值和关键节点的数据标签优先保留，不要把全部标签硬塞进图里。
- 峰值、末值、异常值等重点标签一旦显示，必须带上具体数值本身，不要只显示“峰值”“末值”等抽象字样；其中峰值水滴标签只显示数值本身。必要时增加标签白底、描边、留白或 `symbolSize`，优先保证数值可读。
- ECharts 配置必须显式调整宽度、高度和布局参数，例如 `grid`、`left`、`right`、`top`、`bottom`、`containLabel`、`barGap`、`barCategoryGap`、`axisLabel.rotate`、`axisLabel.margin`、`legend.itemGap`、`legend.padding`，不要依赖默认布局。
- 单文件 HTML 默认直接内嵌 ECharts 配置并在页面脚本中初始化；如果引用 CDN，必须确保在无网络时页面正文与表格仍可阅读，并尽量为图表保留可降级的信息说明。
- 折线图必须先完成时间序列排序、去重与补位校验，再绘制；要保证每个 x 点与对应 y 值严格配对，避免因排序错误、类别错位或缺失补位不一致造成折线漂移。
- 每个图表下方必须配一段斜体的“图表解读”或“数据来源”说明，字号约 `12px`，颜色为 `#666666`。
- 图表下方或上方必须有与图表直接对应的业务解释 bullet，说明峰值、结构特征、节奏特征和管理含义。
- 每个图表必须使用同一个 `.figure-block` 包裹图表标题、图表主体和图表解读，避免打印时标题和图表分离。
- 图表高度建议控制在 `360px` 到 `460px`；如果类目较多，可适度增高或加宽，但仍需保证打印时可读。
- 当数据点很多时，优先增加画布宽度、边距和下方留白，避免轴标签、数据标签和图例重叠。

推荐结构：

```html
<section class="figure-block">
  <h3>每周课表查询趋势</h3>
  <div class="chart-block">
    <!-- ECharts SVG 容器 -->
  </div>
  <p class="chart-note">图表解读：教师课表是主要入口，开学周查询峰值明显。</p>
</section>
```

## 打印分页规则

- 必须添加 `@media print` 样式，尽量避免图表、图表解读、短表格和标题被拆到两页。
- `.figure-block`、`.chart-block`、`.table-block`、短表格、图片、ECharts SVG 容器应设置 `break-inside: avoid; page-break-inside: avoid;`。
- `h2`、`h3` 应设置 `break-after: avoid; page-break-after: avoid;`，避免标题单独落在页尾。
- 正文段落和列表建议设置 `orphans: 3; widows: 3;`，减少孤行。
- 长表格允许自然分页，不要为了禁止分页导致内容溢出页面。
- 打印前建议按 A4、缩放 `90%` 到 `95%`、关闭浏览器页眉页脚进行检查。

推荐打印 CSS：

```css
@media print {
  .figure-block,
  .chart-block,
  .table-block,
  figure,
  img,
  .chart-block svg,
  .echarts-for-react,
  [data-echarts-container] {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  h2,
  h3 {
    break-after: avoid;
    page-break-after: avoid;
  }

  p,
  li {
    orphans: 3;
    widows: 3;
  }

  .chart-block,
  .chart-block svg,
  [data-echarts-container] {
    max-height: 420px;
  }
}
```

## 响应式要求

- 主容器宽度使用 `max-width` 和百分比宽度配合。
- 图片、ECharts SVG 容器、表格包装器不得超出页面宽度。
- 在窄屏下减少左右内边距，保持正文可读。
- 学校概览、报告说明附近的短表和说明块默认按纵向顺序排版，优先保证文档流的从容感；只有当两个区块都足够简短且视觉重量接近时，才考虑并排布局。
