# 数据快照规范

## 作用

`fetch_redash_dashboard.py` 会先把 Redash dashboard 的 query 结果整理成单个 JSON 快照。AI 生成 HTML 时只能读取这个快照，不要重新访问 Redash。

快照是内部数据准备层，不是最终报告的写作口径。最终 HTML 可以使用快照中的信息做内部判断，但不要把 `query` 统计、缓存命中、补取数量、失败数量等技术摘要原样输出给报告读者。

默认路径：

```text
output/school-<school_id>-dashboard-<dashboard_id>-latest.json
```

## 顶层结构

快照必须包含以下字段：

- `generated_at`：快照生成时间，ISO 8601 格式。
- `source`：数据来源信息。
- `dashboard`：dashboard 基本信息。
- `queries[]`：每个 query 的结构化结果。
- `summary`：拉数成功、缓存命中、补取和失败摘要。

示意：

```json
{
  "generated_at": "2026-04-16T15:20:31+08:00",
  "source": {
    "base_url": "https://redash.seiue.com",
    "dashboard_id": "12",
    "school_id": 539
  },
  "dashboard": {
    "id": 12,
    "name": "希悦使用情况",
    "slug": "usage-dashboard",
    "widget_count": 66,
    "query_count": 24
  },
  "queries": [
    {
      "query_id": 40,
      "query_name": "用户信息",
      "status": "cached",
      "fetch_mode": "cached",
      "columns": [],
      "row_count": 3,
      "rows": []
    }
  ],
  "summary": {
    "total_queries": 24,
    "success_count": 24,
    "usable_query_count": 18,
    "cached_count": 15,
    "refreshed_count": 3,
    "empty_count": 4,
    "error_count": 2
  }
}
```

## `source`

- `base_url`：Redash 站点地址，不包含 API key。
- `dashboard_id`：本次使用的 dashboard id。
- `school_id`：本次请求的学校 ID。

## `dashboard`

- `id`：dashboard id。
- `name`：dashboard 名称。
- `slug`：dashboard slug。
- `widget_count`：dashboard widget 数量。
- `query_count`：去重后的 query 数量。

## `queries[]`

每个 query 至少应包含：

- `query_id`
- `query_name`
- `status`
- `fetch_mode`
- `query_result_id`
- `retrieved_at`
- `columns`
- `row_count`
- `rows`
- `error`

### 状态口径

- `cached`：先前已有可用结果，本次直接命中缓存。
- `refreshed`：缓存不可用，本次仅对该 query 单独补取。
- `empty`：query 成功返回但 0 行，不进入报告正文。
- `error`：query 获取失败，不进入报告正文。

`fetch_mode` 用于记录实际获取方式：

- `cached`
- `refreshed`
- 失败时为 `null`

## `summary`

- `total_queries`：dashboard 去重后的 query 总数。
- `success_count`：未报错的 query 数量，包含 `empty`。
- `usable_query_count`：可直接进入正文分析的 query 数量，通常为 `cached + refreshed`。
- `cached_count`：命中缓存数量。
- `refreshed_count`：补取数量。
- `empty_count`：空结果数量。
- `error_count`：失败数量。

## 报告生成使用规则

- 只把 `status` 为 `cached` 或 `refreshed` 且有有效指标的数据集送入正文分析。
- `empty` 和 `error` 不进入目录、正文、图表和表格。
- 如果快照中全部 query 都是 `empty` 或 `error`，停止生成 HTML。
- `summary` 只用于内部判断分析边界、挑选可写模块和识别风险，不作为最终 HTML 的系统提示条或统计摘要原样输出。
- 如需向读者交代分析范围，应改写成自然的分析师口径，例如“本次重点聚焦已有稳定使用记录的核心模块”，不要输出补取数量、失败数量或 query 统计。
