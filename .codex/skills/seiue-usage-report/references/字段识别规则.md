# 字段识别规则

## 识别顺序

1. 先读取脚本输出的 JSON 快照，确认 `source`、`dashboard`、`queries[]` 和 `summary` 完整可用。
2. 按 `queries[]` 中的 `status` 过滤数据集：只把 `cached` 和 `refreshed` 且有有效数据的 query 送入正文分析。
3. 将每个可用数据集记录为：query id、query name、列名、前 5 行样例、行列规模、统计时间范围候选、获取状态。
4. 先过滤无有效数据的数据集，再识别模块；`empty`、0 行、核心指标全空、核心数值全为 0 的数据集不参与报告。
5. 优先按 query name 判断业务主题，再用列名和样例值校验。
6. 对同义字段做有限映射；如果学校名称、统计周期或核心字段仍不明确，先向用户确认。
7. 不把无关业务系统字段强行纳入希悦使用情况分析。

## 可选模块族

学校概览：
- query name 关键词：`用户信息`、`每年级学生人数`、`正式学期`、`学期`
- 常见字段：`角色`、`人数`、`微信绑定率`、`手机绑定率`、`钉钉绑定率`、`学部`、`年级名称`、`届别`、`学生人数`、`学期名称`、`开始日期`、`结束日期`
- 可生成内容：用户规模、绑定情况、学生年级结构、学期时间范围。

活跃趋势：
- query name 关键词：`日活`、`周活`、`活跃`
- 常见字段：`日期`、`学校`、`角色`、`访问人数`
- 可生成内容：日活/周活趋势、角色结构、峰值周或峰值日、开学或考试周期解释。

课程教务：
- query name 关键词：`课程任务`、`选课`、`排课`、`课表`、`调代课`、`课时统计`、`ClassIn`
- 常见字段：`周起始日`、`周开始(周一)`、`周结束(周日)`、`type`、`状态`、`任务布置数`、`任务数`、`选课志愿数`、`snapshot_count`、`week_start`、`类型`、`访问次数`、`调代课次数`
- 可生成内容：作业/资料任务、提交与批阅、选课热度、排课节奏、课表查询、调代课线上化程度、课时统计导出使用。

成绩评价：
- query name 关键词：`过评`、`评价项`、`考试`、`评教`、`评语`、`素养评价`、`教师考核`、`小红花`
- 常见字段：`评价项名称`、`过评项创建数`、`过评项发布数`、`过评分录入数`、`考试创建数`、`考场状态`、`评教提交次数`、`评语任务`、`学期评语数`、`最早创建时间`、`最近创建时间`
- 可生成内容：过程性评价建设、成绩录入节奏、考试与评教应用、学期评语采集。

考勤请假：
- query name 关键词：`考勤`、`请假`、`打卡`
- 常见字段：`周起`、`周止`、`年周`、`行政班考勤班次`、`已考勤课节数`、`类型`、`数量`、`打卡数`、`打卡人次`
- 可生成内容：行政班考勤、课堂考勤、学生/教师请假趋势、教师打卡执行度和常规管理强度。

家校通知与流程协同：
- query name 关键词：`通知`、`家校`、`审批流`、`报表`、`报告单`
- 常见字段：`is_important_count`、`wide_range_count`、`is_var_count`、`all_count`、`周起`、`周止`、`发布人次`、`查看人次`、`导出人次`
- 可生成内容：通知发送量、重要通知和大范围通知使用情况、审批流程线上化程度、报表发布查看与报告单应用。

德育与其他：
- query name 关键词：`德育`、`校历`、`场馆`、`预约`、`里程碑`、`目标管理`、`学籍`、`出入校`、`约谈`、`心理`、`问卷`、`听评课`
- 常见字段：`项目`、`德育分数录入数`、`月份`、`类型`、`校历事件数`、`场地预约发起数`、`name`、`记录数`、`提交数`、`人次`
- 可生成内容：德育评价、校历维护、场馆预约、目标管理、学籍维护、出入校管理、问卷反馈、心理干预与约谈等校务场景。

## 有效数据判断

- `status=empty` 或行数为 0：直接跳过。
- 核心数值列全为空、全为 0 或无法转为可解释指标：直接跳过。
- 只有维度字段、没有数量/比例/时间等可分析指标：直接跳过。
- 查询状态为 `error`、权限不足或脚本未取到结果：不写入报告正文；这些信息仅用于内部判断覆盖边界和风险，不要在最终 HTML 中原样概括失败数量或技术原因。
- 全部 query 均无有效数据：停止，不生成空报告。

## 清洗规则

- 删除全空行和全空列。
- 合并重复列名时保留原始含义，必要时加后缀区分。
- 对明显的合计行、总计行、空模块名行单独处理，不参与模块排名。
- 数值字段中的逗号、百分号、空格和单位要清洗后再计算。
- 百分比字段保留百分比语义，不要与人数或次数直接相加。
- 日期字段需要尝试转换为日期、月份或统计周期；无法转换时不要生成时间趋势。

## 降级策略

- 某个可选模块族没有有效数据：直接跳过，不追问、不写占位段落。
- 某个模块仅有 `empty` 或 `error` query：不进入正文；如确需交代分析范围，仅在“报告说明”中自然表述为“本次重点分析已有明确数据支撑的模块”，不要输出 query 状态或技术统计。
- 有时间和指标：生成趋势分析。
- 有分类字段和指标：生成结构对比或堆叠图。
- 只有汇总指标：生成数据概览、表格和审慎的管理解读。
- 缺少学校名称但数据可分析：输出文件名可使用 `school_id-<ID>`；正文优先使用“该校”“本校”或自然化学校标识，不写 `school_id=<ID>` 这种系统格式。
- 缺少可解释指标或核心字段无法映射：停止并请用户说明字段含义或调整 Redash 查询。
